## 사용자가 본 사건

빈 채팅 화면에 들어왔더니 "🎯 예시 질문" 5개와 "📖 최근 추가된 문서" 6장이 같은 화면에 인접해 있었다. 5개 질문 중 "3D 게임 프로그래밍에서 중요한 기술은 무엇인가요?"라는 질문을, 그 옆의 책 카드 "파이썬 데이터 사이언스 핸드북"에 한정된 sample_question으로 오해했다.

> "파이썬_데이터_싸이언스_핸드북 를 선택 헀는데 3D 게임 프로그래밍에서 중요한 기술은 무엇인가요? 라는 질문이 어떻게 나오게 된거지?"

조사 결과, 실제로는 책을 선택한 적이 없었고 EmptyState의 인덱스 전체 suggested_questions 5개 중 하나였다. 사용자 본인도 헷갈렸으니 외부 사용자에게도 같은 혼란이 가능.

## 첨부 — 실제 데이터

- 책 캐시(`documents.summary -> sample_questions`):
  ```
  ["마라톤 완주 시간 분석은 어떻게 하나요?",
   "이산적인 레이블 예측 모델은 무엇인가요?",
   "NumPy 배열을 생성하는 방법은 무엇인가요?"]
  ```
- 인덱스 전체(`/index/overview` suggested_questions, gpt-4o-mini 1회 호출):
  ```
  - 머신러닝을 혼자 공부하는 방법은 무엇인가요?
  - 파이썬을 이용한 데이터 과학의 기본 개념은 무엇인가요?
  - 클라우드 네이티브 자바의 장점은 무엇인가요?
  - 인공생명에 대한 최신 연구 동향은 어떤 것이 있나요?
  - 3D 게임 프로그래밍에서 중요한 기술은 무엇인가요?  ← 사용자가 클릭
  ```

## 추가 보고 (사용자 정정)

- "책을 선택했을 때 나오는 화면이 고정인 것 같네" → `?doc_filter=` 활성 상태로 채팅 진입해도 EmptyState 화면이 인덱스 전체 그대로
- "이 책에 묻기 하면 나오는 화면은 내가 선택한 책하고는 상관이 없네" → 책 컨텍스트가 EmptyState에 0% 반영

→ 결함 본질: EmptyState가 활성 스코프(doc_filter / category / series_filter) 인지 0. 인덱스 전체 overview만 사용해 어떤 진입 경로에서도 같은 화면이 나옴.
